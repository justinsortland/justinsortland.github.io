## Why Revisit This Paper?

The transformer architecture has dominated NLP and is making inroads into vision and other domains. But viewing transformers through a graph lens reveals deep connections to GNN architectures and suggests interesting hybrid approaches.

## The Graph Perspective

Consider a sequence of $n$ tokens. Self-attention computes:

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

This is equivalent to message passing on a **complete graph** where:
- Each token is a node
- Edge weights are attention scores (computed dynamically)
- Message aggregation uses the softmax-weighted sum

## Key Insight: Dynamic Edge Weights

Traditional GNNs use static edge weights from the input graph. Transformers compute edge weights as a function of node features:

$$
\alpha_{ij} = \frac{\exp(q_i \cdot k_j / \sqrt{d_k})}{\sum_l \exp(q_i \cdot k_l / \sqrt{d_k})}
$$

This allows the model to learn which connections are important for each input.

## Connections to GAT

Graph Attention Networks (GAT) use a similar idea but on sparse graphs:

$$
\alpha_{ij} = \frac{\exp(\text{LeakyReLU}(a^T[Wh_i \| Wh_j]))}{\sum_{k \in \mathcal{N}(i)} \exp(\text{LeakyReLU}(a^T[Wh_i \| Wh_k]))}
$$

The key difference: GAT only attends to graph neighbors, while transformers attend to all tokens.

## Open Questions

1. **Sparse Attention**: Recent work on efficient transformers (Longformer, BigBird) essentially introduces sparsity patterns—learned graph structures. Can we learn the graph structure end-to-end?

2. **Structural Information**: Transformers need positional encodings because the complete graph has no structure. GNNs on real graphs don't need this. Is there a middle ground?

3. **Inductive Bias**: The complete graph attention might be wasteful for structured data. How do we inject the right inductive biases?
