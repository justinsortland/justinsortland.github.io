## Why Build from Scratch?

Libraries like PyTorch Geometric are fantastic, but they abstract away the details. Building a GNN from scratch helped me truly understand what's happening during message passing.

## The Setup

Let's implement the Graph Convolutional Network (GCN) from Kipf & Welling (2017).

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np
```

## Representing Graphs

First, we need to represent our graph. The key data structures:

```python
class GraphData:
    def __init__(self, x, edge_index, y=None):
        """
        x: Node features [num_nodes, num_features]
        edge_index: Edge list [2, num_edges]
        y: Node labels [num_nodes]
        """
        self.x = x
        self.edge_index = edge_index
        self.y = y
        self.num_nodes = x.size(0)
```

The `edge_index` is a 2×E tensor where each column is an edge (source, target).

## Computing the Normalized Adjacency

GCN uses the normalized adjacency matrix:

$$
\tilde{A} = D^{-1/2} (A + I) D^{-1/2}
$$

```python
def compute_normalized_adjacency(edge_index, num_nodes):
    # Add self-loops
    self_loops = torch.arange(num_nodes).unsqueeze(0).repeat(2, 1)
    edge_index = torch.cat([edge_index, self_loops], dim=1)
    
    # Compute degree
    row, col = edge_index
    deg = torch.zeros(num_nodes)
    deg.scatter_add_(0, row, torch.ones(row.size(0)))
    
    # D^{-1/2}
    deg_inv_sqrt = deg.pow(-0.5)
    deg_inv_sqrt[deg_inv_sqrt == float('inf')] = 0
    
    # Build sparse normalized adjacency
    edge_weight = deg_inv_sqrt[row] * deg_inv_sqrt[col]
    
    return edge_index, edge_weight
```

## The GCN Layer

Now the actual layer:

```python
class GCNLayer(nn.Module):
    def __init__(self, in_features, out_features):
        super().__init__()
        self.linear = nn.Linear(in_features, out_features, bias=False)
        self.bias = nn.Parameter(torch.zeros(out_features))
        
    def forward(self, x, edge_index, edge_weight):
        # Transform features
        x = self.linear(x)
        
        # Message passing: aggregate neighbor features
        row, col = edge_index
        out = torch.zeros_like(x)
        
        # For each edge, add weighted source features to target
        for i in range(edge_index.size(1)):
            out[row[i]] += edge_weight[i] * x[col[i]]
        
        return out + self.bias
```

Wait, that loop is slow! Let's use scatter operations:

```python
class GCNLayerOptimized(nn.Module):
    def __init__(self, in_features, out_features):
        super().__init__()
        self.linear = nn.Linear(in_features, out_features, bias=False)
        self.bias = nn.Parameter(torch.zeros(out_features))
        
    def forward(self, x, edge_index, edge_weight):
        # Transform features
        x = self.linear(x)
        
        row, col = edge_index
        
        # Gather source features and weight them
        messages = x[col] * edge_weight.unsqueeze(-1)
        
        # Scatter-add to target nodes
        out = torch.zeros_like(x)
        out.scatter_add_(0, row.unsqueeze(-1).expand_as(messages), messages)
        
        return out + self.bias
```

## The Full Model

```python
class GCN(nn.Module):
    def __init__(self, num_features, hidden_dim, num_classes, dropout=0.5):
        super().__init__()
        self.conv1 = GCNLayerOptimized(num_features, hidden_dim)
        self.conv2 = GCNLayerOptimized(hidden_dim, num_classes)
        self.dropout = dropout
        
    def forward(self, data):
        x = data.x
        edge_index, edge_weight = compute_normalized_adjacency(
            data.edge_index, data.num_nodes
        )
        
        # First layer
        x = self.conv1(x, edge_index, edge_weight)
        x = F.relu(x)
        x = F.dropout(x, p=self.dropout, training=self.training)
        
        # Second layer
        x = self.conv2(x, edge_index, edge_weight)
        
        return F.log_softmax(x, dim=1)
```

## Key Learnings

1. **Message passing is just sparse matrix multiplication** - The normalized adjacency encodes how to aggregate neighbor features

2. **Scatter operations are essential** - Avoid Python loops for graph operations

3. **Normalization matters** - Without proper normalization, feature scales explode or vanish

4. **Self-loops are important** - They ensure a node's own features are included in the aggregation
