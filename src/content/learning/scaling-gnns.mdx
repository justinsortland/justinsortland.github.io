## The Challenge

Most GNN tutorials use tiny graphs (Cora: 2.7K nodes). Real-world graphs are massive:
- Twitter social graph: 1B+ nodes, 100B+ edges
- Web graph: 100B+ pages
- Molecular databases: Billions of molecules

How do we train on these?

## Memory Analysis

For a graph with $n$ nodes, $m$ edges, feature dimension $d$:

| Component | Memory |
|-----------|--------|
| Node features | $O(nd)$ |
| Adjacency (dense) | $O(n^2)$ |
| Adjacency (sparse) | $O(m)$ |
| Hidden states | $O(nd_{hidden})$ |
| Gradients | $2 \times$ above |

For 1B nodes with 100-dim features in float32: 400GB just for features!

## Approach 1: Mini-batch Sampling

Instead of full-batch training, sample subgraphs.

### Node-wise Sampling (GraphSAGE)

For each target node, sample fixed-size neighborhoods:

```python
def sample_neighbors(edge_index, target_nodes, num_samples):
    """Sample neighbors for each target node"""
    sampled_edges = []
    
    for node in target_nodes:
        neighbors = get_neighbors(edge_index, node)
        if len(neighbors) > num_samples:
            neighbors = random.sample(neighbors, num_samples)
        for n in neighbors:
            sampled_edges.append((n, node))
    
    return sampled_edges
```

The **neighbor explosion problem**: for $K$ layers with $S$ samples per layer, we need $S^K$ nodes!

### Layer-wise Sampling (FastGCN)

Sample $s$ nodes per layer instead of per-node:

```python
def layer_sampling(graph, target_nodes, num_layers, samples_per_layer):
    layers = [target_nodes]
    
    for _ in range(num_layers):
        # Sample from all 1-hop neighbors
        candidates = get_all_neighbors(graph, layers[-1])
        sampled = importance_sample(candidates, samples_per_layer)
        layers.append(sampled)
    
    return layers
```

### Subgraph Sampling (ClusterGCN)

Pre-partition the graph, train on subgraphs:

```python
def cluster_gcn_training(graph, num_clusters, epochs):
    # Pre-compute clusters using METIS
    clusters = metis_partition(graph, num_clusters)
    
    for epoch in range(epochs):
        random.shuffle(clusters)
        
        for cluster_batch in batch(clusters, batch_size=10):
            # Extract subgraph induced by cluster nodes
            subgraph = extract_subgraph(graph, cluster_batch)
            
            # Standard GNN forward/backward
            loss = model(subgraph)
            loss.backward()
            optimizer.step()
```

## Approach 2: Feature Propagation

Decouple feature propagation from learning (SGC, SIGN):

```python
def precompute_features(adj, features, num_hops):
    """Precompute multi-hop aggregated features"""
    propagated = [features]
    current = features
    
    for _ in range(num_hops):
        current = adj @ current  # Sparse matmul
        propagated.append(current)
    
    return torch.cat(propagated, dim=1)

# Now training is just an MLP!
mlp_features = precompute_features(adj, x, K)
predictions = mlp(mlp_features)
```

Pros: Very fast training, easy to scale
Cons: No feature transformation between propagation steps

## Approach 3: Distributed Training

For truly massive graphs, distribute across machines.

### Data Parallelism
Each worker has full graph, different mini-batches:
```
Worker 0: nodes 0-999
Worker 1: nodes 1000-1999
...
All-reduce gradients after each step
```

### Graph Parallelism
Partition graph across workers:
```
Worker 0: partition 0 (nodes 0-999M)
Worker 1: partition 1 (nodes 1000M-1999M)
...
Exchange boundary node features each layer
```

```python
def distributed_message_passing(local_features, partition_info):
    # Send boundary features to neighbors
    for neighbor_partition in partition_info.neighbors:
        send_features(
            local_features[partition_info.boundary_nodes],
            neighbor_partition
        )
    
    # Receive boundary features
    remote_features = {}
    for neighbor_partition in partition_info.neighbors:
        remote_features[neighbor_partition] = receive_features()
    
    # Aggregate including remote features
    return aggregate_with_remote(local_features, remote_features)
```

## Practical Recommendations

| Graph Size | Recommended Approach |
|------------|---------------------|
| < 100K nodes | Full batch |
| 100K - 10M | Mini-batch sampling |
| 10M - 1B | Cluster-GCN or SIGN |
| > 1B | Distributed training |

## Memory Optimization Tips

1. **Use half precision**: 2x memory savings
2. **Gradient checkpointing**: Trade compute for memory
3. **Lazy loading**: Stream features from disk
4. **Sparse operations**: Never materialize dense adjacency

```python
# Example: memory-efficient sparse aggregation
def sparse_aggregate(x, edge_index):
    row, col = edge_index
    return scatter_add(x[col], row, dim=0, dim_size=x.size(0))
```
