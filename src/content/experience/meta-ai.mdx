## About the Role

During my internship at Meta AI, I joined the Graph ML Infrastructure team responsible for the systems powering friend suggestions, content ranking, and integrity detection across Meta's family of apps.

## Key Contributions

### Inference Optimization

The social graph contains billions of edges, and GNN inference needs to happen in real-time for user-facing features. I focused on:

1. **Operator Fusion**: Combined multiple sparse-dense operations into single CUDA kernels
2. **Graph Batching**: Developed algorithms to batch structurally similar subgraphs
3. **Embedding Cache**: Implemented distributed caching with intelligent prefetching

### Impact Metrics

| Metric | Before | After | Improvement |
|--------|--------|-------|-------------|
| P50 Latency | 12ms | 6.6ms | 45% |
| P99 Latency | 45ms | 28ms | 38% |
| GPU Utilization | 62% | 84% | 35% |

### A/B Testing Framework

Built infrastructure for testing different graph sampling strategies in production:

- Deterministic sampling for reproducibility
- Automatic metric collection and significance testing
- Integration with internal experiment management system

## Learnings

This internship taught me the importance of systems thinking in ML. The most elegant algorithm is useless if it can't run at scale. I learned to think about cache hierarchies, network topology, and GPU memory management as first-class concerns in ML system design.
