## About the Role

Joined as one of the first three engineers at a startup building a graph database specifically designed for ML workloads. Traditional graph databases weren't built with ML in mind—our goal was to make GNN training as simple as writing a SQL query.

## Technical Contributions

### Query Language Design

Extended Cypher with ML-specific constructs:

```cypher
-- Define a GNN feature extraction
CREATE FEATURE user_embedding AS
  AGGREGATE(
    NEIGHBORS(user, 2),
    method='mean',
    features=['age', 'activity_score']
  );

-- Use in a query
MATCH (u:User)-[:FOLLOWS]->(f:User)
WHERE u.user_embedding <-> f.user_embedding < 0.5
RETURN u, f;
```

### Incremental Feature Maintenance

When graphs change, features need to update. Built a system for incremental updates:

- **Change detection**: Track which nodes/edges were modified
- **Dependency graph**: Understand which features depend on which data
- **Minimal recomputation**: Only update affected features

### Performance

Achieved competitive performance with specialized systems:

| Operation | Neo4j | Our DB | Speedup |
