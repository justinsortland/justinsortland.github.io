## Overview

GraphForge addresses the scalability challenges in training Graph Neural Networks on massive graphs. Traditional approaches load entire graphs into memory, which becomes infeasible for billion-edge networks common in social media, knowledge graphs, and molecular datasets.

## Key Features

### Hierarchical Graph Partitioning

We implement METIS-based partitioning with overlap regions to minimize communication during distributed training:

```python
from graphforge import partition_graph

partitions = partition_graph(
    graph,
    num_parts=8,
    overlap_hops=2,
    balance_factor=1.05
)
```

### Memory-Efficient Message Passing

Custom CUDA kernels for fused message aggregation reduce memory footprint:

$$
h_v^{(l+1)} = \sigma\left(W^{(l)} \cdot \text{AGGREGATE}\left(\{h_u^{(l)} : u \in \mathcal{N}(v)\}\right)\right)
$$

### Distributed Training

Seamless integration with Ray for multi-node training:

```python
trainer = GraphForgeTrainer(
    model=GATv2(hidden_dim=256, heads=8),
    backend="ray",
    num_workers=16
)
trainer.fit(data, epochs=100)
```

## Architecture

The library is built around three core abstractions:

1. **GraphStore**: Lazy-loading graph storage with LRU caching
2. **Sampler**: Configurable neighborhood sampling strategies
3. **Aggregator**: Optimized message-passing primitives
